# 요구사항 (Requirements)

> 출처: `pokemon-random-party-maker-prd.md` (v1 Draft)

---

## 1. 개요

전국도감(1000+ 종)의 포켓몬으로 랜덤 파티(6마리)를 생성하는 단일 페이지 웹 앱.  
**표준 메가진화** 및 **"Z" 메가진화** (예: Mega Lucario Z, Mega Absol Z — *Pokémon Legends Z-A*에서 도입된 신규 서브카테고리) 포함.  
포켓몬 이미지·스탯·타입 데이터는 모두 PokéAPI에서 라이브 fetch — 로컬 이미지 파일 없음.

---

## 2. 목표

- PokéAPI에서 전체 도감(모든 세대, 메가진화, 지역 폼, Z무브 관련 항목)을 동적으로 불러온다. 수동 이미지 없음.
- 한 클릭으로 랜덤 6마리 파티를 생성한다.
- 의미 있는 경쟁·테마 필터를 지원한다.
- 특정 포켓몬을 핀/락하고 나머지 슬롯만 랜덤으로 채운다.
- 생성된 파티의 타입 균형을 자동 분석한다.
- 모듈형·클린·현대적인 코드베이스 및 UI.

### 비목표 (v1 스코프 밖)

- 배틀 시뮬레이션 또는 데미지 계산
- 유저 계정 / 파티 저장 (로컬 스토리지 "즐겨찾기"는 선택적 stretch goal)
- 기술 배치 / EV / IV 빌딩
- 모바일 네이티브 앱

---

## 3. 데이터 소스 & 통합

- **API:** PokéAPI REST (`/pokemon`, `/pokemon-species`, `/evolution-chain`, `/type`, `/pokemon-form`)
- **이미지:** API 반환 `sprites.other["official-artwork"].front_default` URL 사용 — 로컬 저장 없음
- **메가진화·지역 폼:** 별도 `pokemon` 항목으로 제공 (예: `charizard-mega-x`). `formType: "mega"` 태그.
- **Z 메가진화:** `formType: "mega-z"` 태그. PokéAPI 미지원 시 `src/api/patches/megaZ.patch.json` 오버라이드 사용. 동일 스키마이므로 API 지원 시 패치 파일만 삭제하면 됨.
- **캐싱:** 최초 로드 시 전체 목록 1회 fetch 후 메모리/IndexedDB에 캐시. 매 랜덤화마다 재요청 없음.
- **레이트 리밋:** 요청 배치 처리 + 적극적 캐싱.

---

## 4. 핵심 유저 플로우

1. 앱 로드 → 전체 포켓몬 데이터셋 fetch/빌드 (로딩 상태/진행 표시기 표시)
2. 유저가 필터 설정 (지역 제외, 진화 단계, 메가/전설 규칙)
3. 유저가 0–5마리 핀
4. **"파티 생성"** 클릭
5. 앱이 나머지 슬롯을 필터 + 타입 균형 로직으로 랜덤화
6. 결과 화면: 6마리 카드(아트워크, 이름, 타입, 폼 뱃지) + **타입 커버리지 요약** (레이더/바 차트)
7. 핀되지 않은 슬롯 개별 재롤 또는 전체 파티 재롤 가능

## 5. 기능 요구사항

### 5.1 전국도감 풀

- 모든 세대 종 로드. 메가진화, 지역 폼(알로라/가라르/히스이/팔데아), 전설/환상 포함.
- 각 항목 태그: `id`, `name`, `types[]`, `generation`, `region`, `isLegendary`, `isMythical`, `isMega`, `isBaby`, `evolutionStage` (basic/stage1/stage2/final), `canZMove`, `spriteUrl`

### 5.2 필터: 파티당 정확히 1개의 메가 슬롯

- 토글: "메가 1슬롯 보장" — 표준 메가진화, 전설 메가진화, Z 메가진화 중 어느 것이든 허용. 서브 체크박스로 세부 카테고리 한정 가능.
- 활성화 시: 6슬롯 중 1개를 `formType` in `["mega", "mega-z"]` 풀에서 뽑음. 나머지 5개는 일반 풀.
- 비활성화 시: 메가가 자연 뽑기에 나올 수 있음 (선택적 서브토글: 완전 비허용 가능).

### 5.3 필터: 지역별 전설 제외

- 다중 선택 지역 목록 (관동, 성도, 호연, 신오, 하나, 카로스, 알로라, 가라르, 팔데아 등)
- 선택된 지역의 전설/환상 포켓몬 제외

### 5.4 필터: 진화 단계 토글

- "미진화(기본) 폼 포함" On/Off
- "완전 진화 폼 포함" On/Off
- 최소 1개 단계는 활성화 상태여야 함; 모두 비활성화 시 경고

### 5.5 핀/락 기능

- 0–5마리 핀 가능. 나머지 슬롯은 필터 하에서 랜덤.
- 재롤 시 핀된 슬롯은 교체하지 않음.

### 5.6 자동 타입 균형 분석

- 타입 중복이 적고 공격/방어 커버리지가 넓도록 파티 생성 시 균형 패스 실행.
- 접근법: 미대표 타입에 가중치 부여 → 파티 완성 후 타입 커버리지 리포트 계산 → 유저에게 표시 (차트 또는 표).
- 핀된 포켓몬 기준으로 나머지 슬롯에만 균형 적용.

## 6. 데이터 모델 / 용어 기준

PokéAPI / Bulbapedia 컨벤션 준수:
- 18-타입 차트 및 이중 타입
- 표준 지역명 및 세대 그룹
- 공식 전설 vs. 환상 구분
- 공식 메가진화 명명 (`-mega`, `-mega-x`, `-mega-y` 접미사)

## 7. UI/UX & 시각 디자인 가이드라인

- **미학:** 현대적·클린·미니멀 — 카드 기반 레이아웃, 소프트 그림자, 둥근 모서리, 일관된 타입 컬러 팔레트 (표준 포켓몬 타입 색상), 다크 모드 친화적.
- **레이아웃:**
  - 좌/상단 패널: 필터 컨트롤 (모바일에서 접을 수 있음)
  - 중앙: 파티 디스플레이 (6카드 반응형 그리드)
  - 핀 검색 바: 자동완성 + 섬네일 미리보기
  - 파티 하단: 타입 균형 요약 차트
- **모션:** 카드 공개/재롤 시 미묘한 전환 (플립 또는 페이드). 과도한 애니메이션 없음.
- **반응형:** 모바일 퍼스트, ~360px 폭까지 작동.
- **접근성:** 타입 색상 이외 충분한 색상 대비 (아이콘/레이블), 키보드 탐색 가능 필터.

---

## 8. 비기능 요구사항

- 포켓몬 이미지 로컬 저장 없음 — API URL만 사용.
- 데이터셋은 캐시하여 파티 재생성 시 네트워크 재요청 없음.
- 코드베이스는 단일 책임 원칙의 모듈형 구조 유지. ES2020+, async/await.
- 데이터셋 로드 후 파티 생성 < 100ms.

---

## 9. 열린 질문 (Open Questions)

1. PokéAPI에 Z 메가진화 항목이 추가되었는지 구현 시점에 확인. 없으면 `megaZ.patch.json`에 최소 Mega Lucario Z, Mega Absol Z 수동 입력.
2. 지역 폼(알로라/가라르 등)을 "지역" 제외 필터에 포함할지 여부.
3. 나중에 파티 저장/공유 기능(공유 링크, 이미지 내보내기)을 추가할지 여부.
4. 초기 빌드에서 React/TypeScript 대 바닐라 모듈형 JS 선택.
